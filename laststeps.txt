Consolidation applied locally

The repo now has two workflow files:

1. .github/workflows/publish-blog.yml

Workflow name: Site Maintenance

This file now owns both content maintenance jobs:

- publish_blog: publishes generated blog artifacts.
- refresh_youtube: refreshes assets/youtube-latest.json.

Triggers:

- schedule: 17 5,17 * * * for blog publishing.
- schedule: 17 0,12 * * * for YouTube refresh.
- push to main for blog/build-source paths only.
- workflow_dispatch with task choice: blog or youtube.

Consolidation details:

- The old Refresh YouTube Snapshot workflow was merged into Site Maintenance.
- Job-level conditions decide which scheduled task runs.
- Job-level concurrency keeps publish-blog and youtube-refresh separated.
- Manual dispatch now chooses the task instead of needing separate workflows.

2. .github/workflows/quality.yml

Workflow name: Site Quality

This file now owns quality checks and optional manual Pages artifact deploy.

Triggers:

- pull_request to main.
- no automatic direct main push trigger; direct main publishing is handled by Site Maintenance and legacy Pages.
- workflow_dispatch with deploy_pages boolean.

Quality job still runs:

- npm run build:public
- npm run build:check
- npm run test:blog
- npm run perf:budget
- npm run perf:images
- npm run test:consent
- npm run perf:lighthouse

Optional manual Pages deploy:

- If workflow_dispatch deploy_pages is true, Site Quality runs quality first.
- After quality passes, it builds/uploads the Pages artifact and deploys it.
- This preserves the manual artifact deploy functionality without keeping a separate Deploy Pages Artifact workflow.
