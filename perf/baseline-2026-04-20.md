## RUM (GA4) setup

The `web-vitals-beacon.js` script pushes `event: web_vital` with params:
`metric_name, metric_value, metric_delta, metric_id, metric_rating,
nav_type, page_path`. Load the built-in GA4 report:

1. GA4 → Explore → Blank exploration.
2. Dimensions: `Event name = web_vital`, custom dim `page_path`,
   `metric_name`.
3. Metrics: `Event count`, `Avg metric_value`.
4. Break by `metric_name` × `page_path` to spot regressions per route.

The metric values are integers. CLS is multiplied by 1000 (so `12` means
`0.012`). Everything else is milliseconds.

## Regression gates

- **Size:** `npm run perf:budget` — fails on any tracked asset >10% over
  its recorded baseline. Update the baseline intentionally via
  `npm run perf:budget:update` AFTER a perf review.
- **RUM:** If the GA4 p75 for LCP or INP on `page_path = /` or
  `page_path = /standings/` regresses by >15% week-over-week, that is a
  release blocker for any further perf phase.

## Rollback

- Remove the `<script defer src="/scripts/perf/web-vitals-beacon.js">` tags
  from the 4 landing pages + template.
- Delete `scripts/perf/`, `perf/`, and the `perf:*` entries in
  `package.json` scripts.
- No user-facing behavior was changed; no data migrations; no SW changes.
